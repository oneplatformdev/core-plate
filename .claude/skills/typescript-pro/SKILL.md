# TypeScript Pro

Use this skill for any repository with meaningful TypeScript usage.

## Principles

- prefer explicit types at module boundaries
- use inference inside local implementation when it stays readable
- model nullable and async states explicitly
- keep public types stable and discoverable

## Apply When

- designing DTOs, props, service contracts, and API responses
- tightening unsafe `any` usage
- creating reusable hooks, utilities, and test helpers
- refactoring shared types across server and client
