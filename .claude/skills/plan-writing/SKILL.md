# Plan Writing

## Principles

- split work by risk and validation boundary
- keep steps outcome-oriented
- include explicit verification steps
- re-plan when new information invalidates the current path

## Apply When

- the task spans multiple files, layers, or tools
- architecture choices must be decided before coding
