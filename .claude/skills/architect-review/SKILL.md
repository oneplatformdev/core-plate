# Architect Review

## Principles

- clear ownership beats clever abstraction
- shared modules need clear consumers and boundaries
- framework conventions should drive structure unless they block delivery
- accidental monoliths form through cross-layer leakage

## Apply When

- reviewing module design
- splitting domains or refactoring architecture
- deciding whether to introduce a new shared layer
