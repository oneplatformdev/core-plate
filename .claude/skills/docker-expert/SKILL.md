# Docker Expert

## Principles

- keep images reproducible and minimal
- separate build and runtime stages
- mount source intentionally in dev, not by accident in prod
- make health checks and entrypoints explicit

## Apply When

- writing Dockerfiles
- configuring compose stacks
- packaging Next, Nest, or Vite build artifacts
