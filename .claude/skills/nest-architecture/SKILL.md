# Nest Architecture

## Principles

- group code by feature module
- keep controllers thin and services focused
- validate external input with DTOs and pipes
- isolate infrastructure adapters from application logic

## Apply When

- creating or refactoring Nest modules
- adding guards, interceptors, pipes, and exception filters
- designing service and provider boundaries
- structuring background jobs or message consumers
