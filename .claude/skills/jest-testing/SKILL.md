# Jest Testing

## Principles

- mock at system boundaries, not everywhere
- verify contracts, branching behavior, and failures
- keep tests isolated and explicit about setup
- use integration tests where unit tests would overfit implementation

## Apply When

- testing Nest services and controllers
- validating exception paths and DTO behavior
- maintaining existing Jest-based test suites
